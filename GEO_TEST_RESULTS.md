## 시사점

### 1. 키워드 밀도 > 모든 것
- **G가 압도적 승리**: `제주시관악구송도동` (띄어쓰기 없이 붙임)
- 쿼리 `제주시 관악구 송도동`과 정확히 매칭됨
- 띄어쓰기 없이 붙인 형태가 더 효과적일 수 있음

### 2. 이미지 소스 위치가 중요할 수 있음
| 이미지 | src | 선택 |
|-------|-----|:----:|
| A | `https://images.unsplash.com/...` (외부) | ❌ |
| B~H, ★ | `/images/...` (내부) | ✅ (일부) |

**가설: ChatGPT가 같은 도메인 이미지를 선호?**

### 3. 해상도가 영향 있음
| 이미지 | Width | 선택 |
|-------|-------|:----:|
| C | 2400px | ✅ |
| G | 1800px | ✅ |
| H | 1920px | ✅ |
| D | 800px | ❌ |

**D (저해상도)는 한 번도 선택 안 됨**

### 4. Schema.org는 약함
- B, ★ 둘 다 Schema 있지만 선택 안 됨
- H만 선택된 건 Schema보다 **alt text 덕분**일 가능성

### 5. 영어 alt text는 불리
- B, E 둘 다 영어 alt → 선택 안 됨
- F는 영어지만 선택됨 (미스터리 - 페이지 컨텍스트 때문?)

### 6. HTML Priority/Loading Hints 무관
- A가 `priority: true`, `fetchPriority: "high"` 인데도 선택 안 됨
- 브라우저 힌트는 ChatGPT 크롤러에 영향 없음

## 주변 컴포넌트 영향 분석

### 페이지 구조
```
page.tsx
├── Hotel Schema.org (광역) ──────────────────┐
├── Header                                    │
├── PhotoGrid ─────────────────────────────── │ ← 이미지들 여기
│   ├── ImageObject Schema (A, B, H, ★)       │
│   └── 9개 이미지 (다양한 wrapper)            │
├── BookingSidebar                            │
├── Description (제주시 관악구 송도동 언급)    │
└── Footer                                    │
```

### 영향 가능성 있는 요소

#### 1. 광역 Hotel Schema (page.tsx:13-38)
```json
{
  "@type": "Hotel",
  "name": "제노바 스카이 호텔",
  "address": { "addressLocality": "제주시 관악구 송도동" },
  "image": [Unsplash URL]
}
```
- 페이지 전체 컨텍스트 설정
- `image` 필드에 Unsplash URL 있지만 이게 선택되진 않음

#### 2. H1 제목 (page.tsx:68)
```html
<h1>제노바 스카이 호텔 | 제주시 관악구 송도동의 럭셔리 오션뷰</h1>
```
- "제주시 관악구 송도동" 키워드 포함
- 이미지 선택에 영향?

#### 3. 본문 텍스트 (page.tsx:138-144)
```
제노바 스카이 호텔은 제주시 관악구 송도동 해안가에 위치한...
```
- 키워드 반복
- 주변 텍스트 컨텍스트가 이미지 관련성 판단에 영향?

#### 4. 이미지 Wrapper 구조
| Wrapper | 이미지 | 선택 |
|---------|-------|:----:|
| `<figure>` | A, B, G, H, ★ | G✅ H✅ |
| `<picture>` | C | ✅ |
| `<article>` | E | ❌ |
| `<div>` | D, F | F✅ |

- `<article>` wrapper인 E만 선택 안 됨 (영어 alt 때문일 수도)
- Wrapper 자체보다 **alt text가 더 중요**해 보임

#### 5. Caption 텍스트
| 이미지 | Caption | 선택 |
|-------|---------|:----:|
| A | "제주시 관악구 송도동 제노바 스카이 호텔의..." | ❌ |
| G | "제주시 관악구 송도동 탁 트인 바다와..." | ✅ |
| H | "Award-winning luxury suite design" | ✅ |

- Caption에 키워드 있어도 A는 선택 안 됨 (외부 URL 때문?)
- H는 영어 caption인데 선택됨 (alt text가 한글이라서?)

## ChatGPT 시스템 프롬프트 분석

### 이미지 선택 기준 (공식)
1. **Relevance** - 컨텐츠와 직접 관련된 이미지만 포함
2. **Quality** - 고해상도, 선명한 이미지 선호
3. **Accurate Representation** - 이미지가 컨텐츠를 정확히 대표
4. **Diversity** - 중복/유사 이미지 제외

### 시사점
- **Relevance**: G의 alt text가 쿼리와 직접 매칭되어 선택됨
- **Quality**: C(2400px) 선택, D(800px) 탈락 설명
- **Accurate Representation**: F(`hotel room`)가 페이지 컨텍스트와 일치해서 선택

### Bing 이미지 검색 가설
- 시스템 프롬프트에 `image_query`가 Bing 검색 사용한다고 되어있음
- 하지만 실제로 Bing에 우리 이미지 안 뜸 → **이 가설은 틀렸을 가능성**
- ChatGPT 자체 크롤러 (`GPTBot`, `ChatGPT-User`)가 직접 페이지 방문하는 것으로 추정
